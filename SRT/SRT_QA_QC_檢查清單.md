# SRT 字幕檔案 QA/QC 品質檢查清單

## 文件說明
本檢查清單用於驗證和校核SRT字幕檔案的規範性和品質。根據SubRip格式官方規範和行業標準編製。

---

## 第一部分：SRT格式基本規範

### 1.1 檔案基本設定

#### 1.1.1 檔案擴展名
- [ ] 檔案以 `.srt` 為副檔名
- [ ] 檔案名稱使用英文或數字，避免特殊字符

#### 1.1.2 文字編碼
- [ ] 推薦使用 UTF-8 編碼（不帶BOM）
- [ ] 如使用其他編碼，需統一記錄並在交付文件中說明
- [ ] 確保編碼與播放器相容

#### 1.1.3 換行符設定
- [ ] 使用統一的換行符格式（LF 或 CRLF，建議使用 LF）
- [ ] 所有段落間使用一致的換行方式

---

## 第二部分：SRT內容結構規範

### 2.1 序號規範

#### 2.1.1 編號格式
- [ ] 序號從 1 開始，依次遞增
- [ ] 序號為純數字，無空格或特殊字符
- [ ] 每個字幕段落編號唯一，不重複

#### 2.1.2 序號連續性
- [ ] 檢查序號是否連續（無遺漏或跳號）
- [ ] 確認序號與實際字幕順序一致

### 2.2 時間碼規範

#### 2.2.1 時間碼格式
- [ ] 格式：`HH:MM:SS,mmm --> HH:MM:SS,mmm`
- [ ] 小時、分鐘、秒使用兩位數字（00-59）
- [ ] 毫秒使用三位數字（000-999）
- [ ] 使用逗號 `,` 作為小數分隔符（非句號）
- [ ] 使用 `-->` 作為開始和結束時間的分隔符（前後各一個空格）

#### 2.2.2 時間碼邏輯
- [ ] 開始時間 < 結束時間
- [ ] 時間碼不為負數
- [ ] 時間碼按遞進順序排列（後面的字幕開始時間 ≥ 前面的結束時間）
- [ ] 避免過度重疊（如需重疊，應在字幕編輯規範中說明）

#### 2.2.3 時間碼精度
- [ ] 毫秒精度符合音視訊同步需求
- [ ] 避免毫秒值為 999（應為 000，時間進位）

#### 2.2.4 時間碼特殊情況
- [ ] 檢查是否有極端時間值（如超過影片總時長）
- [ ] 確保字幕出現時間符合對白發生時刻
- [ ] 持續時間合理（非過短或過長）

### 2.3 字幕文本規範

#### 2.3.1 文本內容
- [ ] 每個字幕段落至少包含一行文本
- [ ] 文本內容準確，無漏字、錯字、誤譯
- [ ] 確認中文繁簡用法統一（繁體或簡體保持一致）
- [ ] 標點符號使用正確（中文標點 vs 英文標點）

#### 2.3.2 文本長度
- [ ] 每行文本不超過推薦長度（通常 30-42 字元，依顯示設定調整）
- [ ] 確保文本在一般字幕顯示時間內可讀
- [ ] 長段落應合理換行，不應超過 2-3 行

#### 2.3.3 特殊字符
- [ ] 確保所有特殊字符正確顯示
- [ ] 保留必要的標點或符號（如引號、括弧等）
- [ ] 避免控制字符或不可見字符

### 2.4 結構分隔

#### 2.4.1 段落分隔
- [ ] 每個字幕段落由序號、時間碼、文本組成
- [ ] 段落與段落之間必須有一個完全空行分隔
- [ ] 空行不應包含任何字符（包括空格）
- [ ] 檔案末尾最後一個字幕後應有空行（可選但推薦）

#### 2.4.2 結構完整性
- [ ] 無殘留或破損的結構
- [ ] 序號、時間碼、文本三者對應正確

## 第三部分：格式擴展和選項檢查

### 3.1 HTML標籤支援（非官方但廣泛支援）

#### 3.1.1 標籤格式
- [ ] 粗體：`<b>文本</b>` （使用正確）
- [ ] 斜體：`<i>文本</i>` （使用正確）
- [ ] 下劃線：`<u>文本</u>` （使用正確）
- [ ] 字色：`<font color="顏色名稱或十六進位碼">文本</font>` （使用正確）

#### 3.1.2 標籤規範
- [ ] 開啟和關閉標籤配對正確
- [ ] 標籤嵌套邏輯正確（無重疊或不合理嵌套）
- [ ] 標籤不包含多餘空格

#### 3.1.3 標籤相容性
- [ ] 確認目標播放器支援所使用的標籤
- [ ] 若使用較少見的標籤，應測試相容性

### 3.2 座標資訊（非官方擴展）

#### 3.2.1 座標格式（如使用）
- [ ] 格式：`X1:位置 X2:位置 Y1:位置 Y2:位置`
- [ ] 座標值為數字，無單位或特殊字符
- [ ] 座標在時間碼同一行，以空格分隔

#### 3.2.2 座標應用
- [ ] 座標邏輯正確（X1 < X2，Y1 < Y2）
- [ ] 座標值在合理範圍內（相對於視訊解析度）

## 第四部分：內容品質檢查

### 4.1 翻譯/本地化品質

#### 4.1.1 語言準確性
- [ ] 翻譯或對白準確無誤
- [ ] 術語使用一致
- [ ] 俚語、習語翻譯自然流暢
- [ ] 化名、專有名詞翻譯一致

#### 4.1.2 文化適應性
- [ ] 文化參考和表達方式適應目標語言
- [ ] 避免生硬直譯

### 4.2 時序準確性

#### 4.2.1 同步檢查
- [ ] 字幕出現時間與對白開始相符
- [ ] 字幕消失時間與對白結束相符
- [ ] 字幕持續時間合理（足以讓觀眾閱讀）

#### 4.2.2 速度檢查
- [ ] 閱讀速度合理（通常 120-180 詞/分鐘）
- [ ] 複雜內容允許更長顯示時間
- [ ] 簡短內容不應過長

### 4.3 格式一致性

#### 4.3.1 樣式一致
- [ ] 相同類型字幕使用相同樣式
- [ ] 顏色使用一致（如有）
- [ ] 文本對齊方式統一
- [ ] 字體樣式（粗體、斜體）使用規則一致

#### 4.3.2 標點符號一致
- [ ] 問號、感嘆號使用一致
- [ ] 省略號格式統一（... 或 …）
- [ ] 引號樣式一致（中文「」或英文 "" 或「」）

### 4.4 可讀性檢查

#### 4.4.1 視覺排版
- [ ] 每行字數合理
- [ ] 段落換行位置自然（斷句合理）
- [ ] 避免孤立詞（單詞獨佔一行）

#### 4.4.2 時間充足
- [ ] 短字幕持續時間 >= 1 秒
- [ ] 中等長度字幕持續時間 >= 2 秒
- [ ] 長字幕持續時間 >= 3 秒或更長

## 第五部分：技術檢查

### 5.1 檔案完整性

#### 5.1.1 檔案大小
- [ ] 檔案大小合理（通常 10KB - 5MB）
- [ ] 無異常大小表現

#### 5.1.2 檔案驗證
- [ ] 檔案能在標準播放器中打開
- [ ] 無損壞或截斷的段落
- [ ] 檔案最後正確結束（無亂碼或殘留資料）

### 5.2 播放器相容性

#### 5.2.1 標準播放器測試
- [ ] 在常見影片播放器測試（VLC、Windows Media Player、MPC-HC 等）
- [ ] 在串流平台測試（YouTube、Vimeo 等）
- [ ] 確保時間碼顯示正確

#### 5.2.2 設備相容性
- [ ] 桌面播放器相容
- [ ] 手機/平板播放器相容（如需要）
- [ ] 電視/機上盒相容（如需要）

### 5.3 錯誤檢測

#### 5.3.1 常見錯誤
- [ ] 無重複的序號
- [ ] 無時間碼順序錯亂
- [ ] 無格式錯誤（時間碼格式、分隔符）
- [ ] 無空行遺漏或多餘

#### 5.3.2 邊界條件
- [ ] 首個字幕時間不為 00:00:00,000
- [ ] 末個字幕時間不超過影片總時長
- [ ] 檢查極端時間值

## 第六部分：交付檢查清單

### 6.1 前置檢查

- [ ] 檔案名稱正確無誤
- [ ] 檔案編碼確認為 UTF-8
- [ ] 檔案大小在合理範圍

### 6.2 內容檢查

- [ ] 完整查看整個檔案（至少抽查 10% 的內容）
- [ ] 檢查開頭、中間、結尾部分的品質
- [ ] 確認無明顯的遺漏或重複

### 6.3 技術檢查

- [ ] 在至少兩個不同播放器上測試
- [ ] 確認所有時間碼有效
- [ ] 驗證所有段落結構正確

### 6.4 交付文件

- [ ] 提供檔案的中文翻譯或說明（如需）
- [ ] 文件應包含版本號和修改日期
- [ ] 如有特殊編碼或格式，應附註說明

## 第七部分：常見問題與解決方案

### 7.1 時間碼問題

| 問題 | 原因 | 解決方案 |
|------|------|--------|
| 時間碼無法識別 | 格式錯誤（使用句號而非逗號，或 `->` 而非 `-->` ） | 檢查並修正分隔符 |
| 字幕同步不佳 | 時間碼精度不足或時差累積 | 調整時間碼或使用同步工具 |
| 時間碼超出範圍 | 超過影片時長 | 縮短字幕持續時間或檢查原始資料 |

### 7.2 編碼問題

| 問題 | 原因 | 解決方案 |
|------|------|--------|
| 中文亂碼 | 編碼不匹配 | 轉換為 UTF-8 編碼 |
| 特殊符號顯示錯誤 | 編碼不支援 | 確認編碼相容性或更換符號 |

### 7.3 結構問題

| 問題 | 原因 | 解決方案 |
|------|------|--------|
| 字幕無法顯示 | 序號、時間碼或文本結構錯誤 | 檢查並修復結構 |
| 播放器無法解析 | 分隔符不正確（缺少空行） | 確保每個段落間有空行 |

## 第八部分：品質評分標準

### 8.1 合規性評分

**A 級（優秀）**
- 所有規範項目合格
- 格式 100% 符合標準
- 無錯誤或不一致

**B 級（良好）**
- 絕大部分規範項目合格
- 格式 95% 以上符合標準
- 輕微錯誤或不一致，但不影響使用

**C 級（可接受）**
- 大部分規範項目合格
- 格式 85% 以上符合標準
- 存在明顯錯誤或不一致，但仍可使用

**D 級（不可接受）**
- 規範項目合格率低於 85%
- 存在嚴重錯誤，影響播放或閱讀

## 附錄：SRT 格式規範參考

### A1. SubRip 官方規範

**基本結構示例：**
```
1
00:00:00,000 --> 00:00:05,000
首個字幕內容

2
00:00:05,500 --> 00:00:10,000
第二個字幕內容
可以有多行

3
00:00:10,100 --> 00:00:15,500
第三個字幕內容
```

### A2. 時間碼計算公式

- 總毫秒 = (小時 × 3,600,000) + (分鐘 × 60,000) + (秒 × 1,000) + 毫秒
- 例：01:30:45,500 = (1 × 3,600,000) + (30 × 60,000) + (45 × 1,000) + 500 = 5,445,500 ms

### A3. 推薦工具

| 工具名稱 | 功能 | 平台 |
|---------|------|------|
| Subtitle Edit | 編輯、驗證、同步 | Windows、Mac、Linux |
| Aegisub | 專業編輯、特效 | Windows、Mac、Linux |
| Jubler | 輕量編輯器 | Windows、Mac、Linux |
| VLC | 播放與基礎編輯 | 多平台 |
| Online SRT Validator | 線上驗證 | Web 平台 |
